Architecture

Two packages. Zero compromise.

Separate the structure from the skin. Rig handles behavior and accessibility. Hex handles aesthetics. Swap, extend, or replace either independently.

Rig @amulet-laboratories/rig

Headless, accessible Vue 3 components. Semantic HTML with data-rig-* attributes. Zero styling opinions. Full keyboard nav and ARIA built in.

Headless TypeScript strict Slot-driven WCAG AAA Tree-shakeable ESM + CJS

Hex @amulet-laboratories/hex

Pure CSS theme layer. Four curated themes targeting Rig's data attributes. Hot-swap by changing one import. Zero JavaScript. Built on Tailwind v4 and CSS custom properties.

CSS-only 4 themes Tailwind v4 Custom properties 22 tokens Domain accents
Usage

Import. Compose. Ship.

Every component is a building block. Named slots give you full control over what renders. Data attributes give you full control over how it looks.

  • Tree-shakeable imports from a single entry point
  • Typed named slots with defineSlots
  • v-model bindings on stateful components
  • Style via data attributes or Hex themes
  • Composables for keyboard, drag-drop, focus traps
App.vue 
<!-- One import for the theme -->
<script setup lang="ts">
import '@amulet-laboratories/hex'
import {
  Button,
  Modal,
  CommandPalette,
  TreeView,
} from '@amulet-laboratories/rig'
import { useKeyboard } from '@amulet-laboratories/rig'

const open = ref(false)
useKeyboard({ 'mod+k': () => (open.value = true) })
</script>

<template>
  <Button variant="primary">Open</Button>

  <CommandPalette
    v-model:open="open"
    :items="commands"
    placeholder="Search commands..."
  />
</template>

Positioning

Built for a different tradeoff.

vs Vuetify

No opinion lock-in

Vuetify bundles styling into the component runtime. Rig ships zero CSS. You own the visual layer entirely — use Hex, Tailwind, or raw CSS.

vs shadcn/ui

A real library, not copy-paste

shadcn copies source files into your project. Rig is an installable, versioned package with 110 components. Updates come from npm, not re-generating files.

vs Radix Vue

Vue-native. Not a port.

Radix Vue ports React patterns to Vue. Rig is designed for Vue from the ground up — defineSlots, v-model, provide/inject.

Principles

What we care about.

Accessibility first

WCAG AAA contrast. Full ARIA attributes. Keyboard navigation on every interactive component. Focus management across modals and overlays.

Separation of concerns

Components have zero styling opinions. Themes have zero behavioral logic. Either can be replaced without touching the other.

Developer ergonomics

TypeScript strict. Typed slots. Single entry point. Tree-shakeable. ESM and CJS. Works with Vite, Nuxt, or any bundler.

Minimal surface area

Two peer dependencies: Vue 3.5+ and Floating UI. One optional dep for icons. No CSS framework required unless you want Hex.

Composition over configuration

Named slots, not prop soup. Render what you want, where you want. Every visual element is overridable.

No magic

Components render semantic HTML with data attributes. What you see in DevTools is what you get. No generated class names, no runtime CSS-in-JS.
